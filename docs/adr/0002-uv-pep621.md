# ADR 0002 — PEP 621 pyproject + uv lockfile

**Status:** accepted (2026-07-13)

**Context:** The project had a one-line requirements.txt and no declared
Python support; installs were not reproducible.

**Decision:** Standard PEP 621 `[project]` metadata with hatchling,
dependency groups for dev/evals, and a committed `uv.lock`. CI installs via
`uv sync` so every run resolves identical versions.

**Consequences:** Runtime deps stay minimal (streamlit only). A one-line
`requirements.txt` exists solely as the Streamlit Community Cloud deploy
manifest (the platform does not read PEP 621/uv metadata — see ADR 0005).
